## About

Project Storyteller is a Claude Code-native CLI tool that transforms any code repository into a suite of publication-ready written artifacts. It is not a Python script that calls an API — it is a set of slash commands and skills that let Claude itself read, understand, and write about your project using its own file tools and reasoning.

Built as a Prompt Engineering course project, it demonstrates multi-stage prompt pipelines, reusable skill modules, subagent coordination, and distinct voice engineering across multiple output types.

---

## Why This Exists

Developers are good at building things. They are often less good at explaining them. A great project with a mediocre README gets ignored. A compelling article about the same project gets shared. A well-structured lecture turns a weekend hack into something teachable.

Project Storyteller solves the communication gap — not by summarizing your code, but by genuinely understanding it and then writing about it in whatever voice the audience needs.

## Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/yourusername/project-storyteller
cd project-storyteller

# 2. Open Claude Code
claude

# 3. Check dependencies
/setup

# 4. Analyze your project
/analyze "path/to/your/repo"

# 5. Generate everything
/all
```

Your output will appear in `./output/` within minutes.

| `/resources` | `output/resources.md` | Curated external resources with learning path via web search |
| `/ask` | Terminal | Ask layered questions about the analyzed codebase |
| `/all` | All of the above | Run the full pipeline in sequence |

## How It Works

Project Storyteller runs a three-phase pipeline on every session:

**Phase 1 — Read**
Claude uses its native file tools to walk the repository, reading in priority order: config files → entry points → core logic → tests. Binary files, dependency folders, and lock files are skipped automatically.

**Phase 2 — Analyze**
Before generating anything, Claude builds a structured mental model of the project across three layers: technical (architecture, patterns, design decisions), human (who built this, who uses it, what problem it pushes back against), and voice (5 tone keywords that differentiate the output of each generator).

**Phase 3 — Generate**
Each command spawns a focused generation task using the analysis as shared context. The `/diagram` command generates real PNG files via the Mermaid.ink API. The `/lecture` command produces a PDF via Pandoc. The `/resources` command uses live web search to find and verify relevant external resources.

## Skills Architecture

Three reusable prompt modules power the generation pipeline:

**`skills/analogy-builder.md`** — Maps technical components to real-world analogies. Used by `/lecture` and `/generate-article` to make complex systems accessible to non-technical readers.

**`skills/curriculum-designer.md`** — Sequences project concepts from simple to complex across four tiers: Foundation, Core, Advanced, Expert. Powers the learning path in `/lecture` and `/resources`.

**`skills/diagram-builder.md`** — Generates valid Mermaid syntax for three diagram types: architecture (flowchart TD), data flow (flowchart LR), and sequence diagrams. Used by `/diagram` and `/lecture`.

## Contributing

Project Storyteller is built entirely from prompt files — no compilation, no build step. Contributing means improving the quality of the prompts.

To propose a new command: add a `.md` file to `.claude/commands/`, test it against at least two different repositories, and open a PR with example outputs.

To improve an existing prompt: edit the relevant `.md` file and document what changed and why in your PR description. Prompt decisions are the source of truth here.
